# Evals

`evals.json` 定义了 3 条评估用例，覆盖 skill 的三个核心使用场景：

| ID | 场景 | 覆盖类型 |
|----|------|----------|
| 1 | 混合类型（GitHub + 小红书 + 公众号） | 项目分析卡 / 社媒分析卡 / 内容洞察卡 |
| 2 | Claude Code 主题，输出到 markdown 文件 | 项目分析卡 / 学习价值卡 / 内容洞察卡 / 视频分析卡 |
| 3 | Prompt 模式（生成喂给 DeepSeek 的 prompt） | 多 Git 仓库 + 产品官网 |

## 最近一次 Benchmark 结果

运行于 iteration-1（Scrapling 重构前基线）：

| 配置 | 平均通过率 | 平均 tokens | 平均耗时 |
|------|-----------|------------|----------|
| **with_skill**     | **94.4%** (20/21) | 57,338 | 287.7s |
| without_skill 裸跑 | 39.5% (8/21)      | 36,926 | 231.9s |

**delta**：通过率 +55 个百分点，token +55%，耗时 +24%。

核心发现：
- skill 带来的关键增益是「A/B/C 报告结构 + 类型分组 + 分析卡模板差异化」，这些在 baseline 中全部缺失
- 「我的判断」和「建议的后续行动」是 skill 独有字段，baseline 完全没有
- Eval 3（prompt 模式）baseline 通过率最高（66.7%），因为 prompt 输出格式本身不依赖模板结构
- Token 开销主要来自读取 SKILL.md、config、good-shots 等额外上下文
- 正文提取降级链（Jina Reader → Playwright）在 eval-1 中成功执行，是 skill 的核心差异化能力

## 断言示例（eval-1）

```json
[
  "报告包含 A(研究总览)、B(分类型逐条整理)、C(研究结论) 三段结构",
  "3 个链接分别使用了不同的分析卡模板",
  "每个条目都包含'我的判断'部分",
  "每个条目都包含'建议的后续行动'",
  "无法访问的链接有醒目标注",
  "分析内容具体可比较，没有空洞的形容词堆砌",
  "链接按类型分组而非按原始顺序排列"
]
```

# 分析卡模板

每种链接类型对应一种分析卡。输出时使用大纲格式（层级标题 + 缩进列表），不要用表格。
以下模板是默认格式，用户可通过 `~/.config/link-researcher/templates/` 下的同名文件覆盖。

---

## 1. 项目分析卡（GitHub / Git 仓库）

```
### {序号}. {项目名}

- **仓库**：{URL}
- **类型**：GitHub / Git 仓库
- **访问状态**：{状态}
- **一句话定位**：{这个项目具体做什么，用一句话说清楚}
- **协议**：{MIT / Apache-2.0 / GPL / AGPL / 未注明}
- **技术栈**：{主要语言、框架、运行环境}
- **项目成熟度**：{根据 Star 数、更新频率、Issue/PR 活跃度、文档完善度综合判断}

#### 核心能力
- {能力 1：具体描述}
- {能力 2}
- {能力 3}

#### 关键特征
- {列出与当前热点相关的特征：Agent / MCP / 自动化 / 浏览器控制 / 多模态 / 工作流 / RAG / 本地部署 / 云端 等}

#### 适用场景
- {场景 1}
- {场景 2}

#### 部署/使用方式
- {本地运行 / Docker / 云部署 / 托管 / npm install / pip install 等}

#### 进入门槛
- {低 / 中 / 高} — {原因，如"需要自建 GPU 环境"或"pip install 即可开箱使用"}

#### 我的判断
- **最值得关注的点**：{具体}
- **局限或风险**：{具体}
- **更适合谁**：{具体人群}

#### 建议的后续行动
- {如"Star + clone 本地跑 demo 试试"、"等 v2.0 稳定后再关注"等}
```

---

## 输出格式要求

- 所有卡片使用大纲模式（标题 + 缩进列表），不用表格
- 中文输出
- 不要复制网页原文或大段宣传语
- 不要只写"这是一个 AI 工具"，要说明它具体做什么、服务谁、与同类相比怎样
- 信息不足时明确写"基于当前可见信息判断"
- 如果链接是某产品的子页面（pricing、settings、教程章节），要说明它在整体产品/生态中的位置

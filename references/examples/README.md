# 示例输出

这里是 link-researcher 技能的真实运行产出，作为报告结构和输出质量的参考标杆。

每一份都按 skill 规定的 A（研究总览）/ B（分类型逐条整理）/ C（研究结论）三段式组织，
按类型分组、组内按研究价值排序，每条都包含「我的判断」和「建议的后续行动」。

## 列表

- [`report-openclaw-agent-ecosystem.md`](./report-openclaw-agent-ecosystem.md) — 17 条链接的 Agent 生态研究，覆盖 Git 仓库 / 产品官网 / 文章 / 社媒多类型混合，含采集降级链说明
- [`report-misc-mixed-batch.md`](./report-misc-mixed-batch.md) — 5 条分散主题的小批量研究，展示「没有集中主题」情况下的结论写法

## 用法

- 想看完整报告长什么样 → 直接读这两份
- 想让 skill 模仿某种风格 → 挑出理想片段放到 `~/.config/link-researcher/good-shots/` 下，按类型命名（如 `github-repo.md`、`article.md`），skill 会把它当作质量标杆
- 想写自定义模板 → 参考这些报告里的字段组织，再改 `~/.config/link-researcher/templates/`

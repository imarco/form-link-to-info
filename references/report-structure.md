# 研究报告结构

报告分三个部分，严格按此顺序输出。

---

## A. 研究总览

这部分让读者 30 秒内了解全貌。

```markdown
# 链接研究报告

## A. 研究总览

| 指标 | 数量 |
|------|------|
| 原始链接总数 | {N} |
| 去重后数量 | {N} |
| 成功访问 | {N} |
| 部分可见 | {N} |
| 无法访问/受限 | {N} |

### 类型分布

- Git 仓库：{N} 个
- 工具/产品官网：{N} 个
- 文章/博客/公众号：{N} 个
- 文档/教程/课程：{N} 个
- 视频：{N} 个
- 社交媒体帖文：{N} 个
- 导航站/聚合平台：{N} 个
- 其他：{N} 个

### 主题聚类

这批链接主要集中在以下领域：
- **{主题 1}**（{N} 个）：{一句话描述这个聚类的特征}
- **{主题 2}**（{N} 个）：...
- ...

主题示例（按实际内容选择）：
Agent / MCP / Prompt Engineering / AI IDE / Browser Automation /
知识库-RAG / 设计工具 / 开发者工作流 / 商业化 / 内容生态 /
社交媒体运营 / 教育资源 / 基础设施 等
```

---

## B. 分类型逐条整理

按以下固定顺序分组输出。空的分组可以跳过。

每个分组使用二级标题，每个链接使用三级标题，组内按研究价值从高到低排序。

```markdown
## B. 分类型逐条整理

## B1. Git 仓库
{使用项目分析卡模板，逐条编号}

## B2. 工具官网 / 产品官网
{使用产品研究卡模板}

## B3. 官方文档 / 教程 / 课程
{使用学习价值卡模板}

## B4. 文章 / 博客 / 公众号 / 视频
{文章使用内容洞察卡，视频使用视频分析卡}

## B5. 导航站 / 聚合平台 / Marketplace
{使用平台观察卡模板}

## B6. 社交媒体帖文
{使用社媒分析卡模板}

## B7. 其他
{使用定制分析卡模板}
```

### 编号规则

全局连续编号，格式为 `{组序号}.{组内序号}`，例如：
- B1.1, B1.2（Git 仓库组内的第 1、2 个）
- B2.1, B2.2（产品官网组内的第 1、2 个）

---

## C. 研究结论

这部分是报告最有价值的部分，要体现研究员的判断力。

```markdown
## C. 研究结论

### 重点关注列表

从全部链接中选出最值得关注的（top 10 或 top 30%，取较小值）：

| 排名 | 名称 | 类型 | 一句话理由 |
|------|------|------|------------|
| 1 | {名称} | {类型} | {为什么值得关注} |
| ... | | | |

### 分维度推荐

- **最值得深挖的 GitHub 项目**：{列出 + 理由}
- **最值得实际体验的产品**：{列出 + 理由}
- **最值得精读的文章/文档/课程**：{列出 + 理由}
- **最值得关注的人/账号**：{如果有社媒链接}

### 趋势观察

这批链接整体反映出的趋势：
- {趋势 1：具体描述}
- {趋势 2}
- {趋势 3}

### 后续研究路径

如果要继续深入研究，建议的下一步：
1. {具体行动 1}
2. {具体行动 2}
3. {具体行动 3}
```

---

## 分批输出规则

如果一次输出装不下全部内容：

1. 第一批先输出 **A. 研究总览**（完整）+ **B 的前几组**
2. 后续批次继续 B 的剩余分组
3. 最后一批输出 **C. 研究结论**
4. 每批开头注明：`> 本批处理链接：第 {X}-{Y} 条（共 {N} 条）`
5. 每批保持相同的分析深度和质量标准
